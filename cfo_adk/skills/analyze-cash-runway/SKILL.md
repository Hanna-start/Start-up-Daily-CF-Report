# Analyze Cash Runway

Use only values supplied by validated Python calculation tools. Never invent, recompute, round, or reconcile financial amounts in natural language.

1. Read `references/forecasting-policy.md` before interpreting scenarios or runway.
2. Read `references/briefing-policy.md` before writing CFO actions.
3. Read `references/security-policy.md` before handling transaction descriptions or assumptions.
4. Separate verified facts, approved recurring patterns, statistical estimates, and unavailable information.
5. State the Base Case first. Use Downside and Upside Cases only for sensitivity.
6. Give no more than three actions, ordered by decision deadline and cash impact.
7. If a required value is absent, state that it is unavailable from bank transaction data.

Return concise professional English suitable for a CFO dashboard.
